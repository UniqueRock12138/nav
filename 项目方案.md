### 1. 项目结构

```
nav/
├── index.html      入口，结构骨架（顶部三个 Tab + 三个内容区）
├── style.css       样式（基础配色、布局、折叠动画）
├── script.js       Tab 切换、递归渲染提示词树、折叠展开、复制
├── data.js         所有数据（links / prompts / communities）
└── README.md       部署说明 + 数据维护说明
```

无构建、无依赖、无 CDN。双击 `index.html` 直接可用，整个目录推到 GitHub Pages 仓库即可。

### 2. 数据规约（`data.js`）

```javascript
const DATA = {
  links: [
    { name: "...", url: "..." }
  ],
  prompts: [
    // 叶子节点：有 content
    { title: "...", content: "..." },
    // 非叶子节点：有 children，可任意嵌套
    { title: "...", children: [ /* 同样结构 */ ] }
  ],
  communities: [
    { name: "...", url: "..." }
  ]
};
```

**硬约束**：每个 prompt 节点 `content` 与 `children` 二选一互斥。`content` 为空字符串 `""` 时视为"待补充"节点，渲染为禁用态。

### 3. 页面交互

**顶层**：三个 Tab —— `链接` / `提示词` / `社区`。点击切换，URL hash 同步（如 `#prompts`），刷新保留当前 Tab。

**链接 Tab**：扁平列表，每行一个链接，点击新窗口打开。

**提示词 Tab**：递归折叠树，进入时全部收起只显示一级标题。

- 非叶子节点：左侧 `▶` / `▼` 指示符，点击展开/收起子节点
- 叶子节点：点击展开/收起 `content`，正文用 `<pre>` 渲染（等宽字体、保留换行、无语法高亮）
- 空 content 节点：灰色禁用态，不可点击，标题后缀 `(暂无)`
- 多节点可同时展开（非手风琴）

**正文区右上角**：固定的"复制"按钮，点击复制 `content` 原文（不含标题、不含任何包裹符）。复制成功后按钮文字临时变为"已复制 ✓"，2 秒后恢复。失败则变为"复制失败"。

**社区 Tab**：与链接 Tab 同结构，独立维护。

### 4. 复制实现

优先 `navigator.clipboard.writeText()`。`file://` 协议下该 API 在部分浏览器不可用，降级到 `document.execCommand('copy')`（已废弃但所有浏览器仍支持，覆盖本地双击场景）。两条路径都失败再提示用户手动复制。

### 5. 样式约束

- 配色：浅灰背景 + 深灰文字 + 单一强调色（如靛蓝 `#4f46e5`）用于 Tab 选中态、链接、按钮
- 字体：系统默认字体栈（`-apple-system, Segoe UI, ...`），代码区用 `ui-monospace, Consolas, monospace`
- 布局：最大宽度 `960px` 居中，左右留白
- 响应式：单一断点 `768px` 以下，Tab 文字缩小、内边距收紧
- 无阴影、无渐变、无动画特效；仅折叠展开用一个 `transition: max-height` 或直接 `display` 切换

### 6. 维护流程

新增链接：在 `data.js` 的 `links` 或 `communities` 数组追加一项。

新增提示词：

1. （可选）先在 `prompt-role.md` 中记录原文，作为人类可读归档
2. 在 `data.js` 的 `prompts` 树中追加节点，content 用反引号模板字符串

注意：模板字符串中如出现反引号 ``` 或 `${` 需要转义为 `\`` 和 `\${`。当前 md 文件扫描结果：未出现这两个序列（高可能，正式动手前再 grep 确认）。

### 7. GitHub Pages 部署

仓库根目录放上述 5 个文件 → 仓库 Settings → Pages → Source 选 `main` 分支根目录 → 等几十秒生效。无需任何配置文件。
